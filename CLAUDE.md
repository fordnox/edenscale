# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common commands

All commands run from the repo root via the `Makefile`:

- `make sync` — install deps (`uv sync` in `apps/backend/`, `pnpm install` for the workspace)
- `make start-backend` — FastAPI dev server on `localhost:8000`
- `make start-manager` / `make start-investor` / `make start-superadmin` — Vite dev servers on ports 3000 / 3001 / 3002 (via `pnpm turbo run dev --filter=<app>`)
- `make start-worker` — run the arq worker (`app.worker.WorkerSettings`)
- `make test` — `pytest -v` in `apps/backend/`
- `make lint` — runs, in order: an import smoke test (`from app import *`), `ruff check --fix`, `ty check`, `black`, `isort`. Tests, `.venv`, and `app/alembic` are excluded from all linters.
- `make openapi` — regenerates `apps/backend/openapi.json` from the FastAPI app, then runs the `generate-client` turbo task in `packages/api/` to update `packages/api/src/schema.d.ts`
- `make migration` — prompts for a name and runs `alembic revision --autogenerate`
- `make upgrade` / `make downgrade` — apply / revert one alembic migration
- `make db-init` — create tables from models (dev only); `make db-seed` — idempotent demo dataset (`scripts/seed_demo`)
- `make kamal-build` / `make kamal-deploy` / `make kamal-logs` — backend deploy via Kamal (config in `config/deploy.yml`, secrets via 1Password `op`)

Run a single backend test: `cd apps/backend && uv run pytest tests/test_api.py::test_name -v`

Frontend type-check only: `cd apps/<manager|investor|superadmin> && pnpm run lint` (`tsc --noEmit`), or `pnpm turbo run lint` for the whole workspace.

### Pre-commit rules (from README.md)
1. `make test` must pass.
2. `make lint` must pass.
3. `make openapi` must be run so the generated client stays in sync.

## Architecture

pnpm + Turborepo monorepo (`pnpm-workspace.yaml`: `apps/*`, `packages/*`). One Python FastAPI backend serves three React SPAs, wired together through a generated OpenAPI client — the frontends never hand-write API types.

### Apps (`apps/`)

- **`backend/`** — Python FastAPI API (see below).
- **`manager/`**, **`investor/`**, **`superadmin/`** — React/Vite SPAs (dev ports 3000/3001/3002). Built with `base: '/manager/'` etc. so they mount under path prefixes on the production domain.
- **`web/`** — Astro marketing site (serves `/`).
- **`docs/`** — Astro Starlight docs site (serves `/docs`).
- **`gateway/`** — Cloudflare Worker for `newtaven.com`. `scripts/assemble.mjs` turbo-builds web + the three SPAs + docs and copies their `dist/` outputs into one asset tree (`/`, `/manager`, `/investor`, `/superadmin`, `/docs`); `worker/index.ts` serves assets with SPA-shell fallback for document navigations only (never for missing hashed assets). Deploy with `pnpm run deploy` (wrangler).
- **`emails/`** — React Email templates (`emails/admin`, `emails/customer`, `emails/layout`). `pnpm run dev` previews on port 4000; `pnpm run push` (`scripts/push-templates.mts`) uploads templates to Resend as hosted templates.
- **`design/`** — NewTaven design system reference (tokens in `colors_and_type.css`, UI kits, brand guidance). Not a build target.

### Shared packages (`packages/`)

- **`@edenscale/api`** — typed `openapi-fetch` client (`src/client.ts`) built from `src/schema.d.ts`, which is generated by `make openapi` — do not edit by hand. Middleware auto-attaches the Hanko session token and surfaces non-401 errors through `sonner` toasts. Also exports TanStack Query helpers (`queryClient`, `useApiQuery`, `useApiMutation`).
- **`@edenscale/auth`** — Hanko web-component auth (`hanko.ts`, `useAuth.ts`).
- **`@edenscale/shared`** — cross-app hooks/contexts (active organization, pending invitations) and formatting utils.
- **`@edenscale/ui`** — shadcn-style components (`src/components/ui/`), shared styles, invitation UI.
- **`@edenscale/brand`**, **`@edenscale/site-config`**, **`@edenscale/typescript-config`** — brand assets, site constants, shared tsconfigs.

### Backend (`apps/backend/app/`)

- **Entrypoint**: `app/main.py` creates the `FastAPI` app, configures CORS from `settings.CORS_ALLOW_ORIGINS`, adds `AuditContextMiddleware`, and mounts ~20 routers (funds, investors, commitments, capital-calls, distributions, documents, communications, tasks, notifications, audit-logs, organizations, invitations, superadmin, users, dashboard, …) each under its own prefix. Auth is declared in each router file via `APIRouter(dependencies=[Depends(get_current_user)])` — not at `include_router` level. Superadmin-only routes additionally use `Depends(require_superadmin)`. OpenAPI/docs URLs are only exposed when `GENERATE_OPENAPI_DOCS` is set.
- **Layered structure**: `routers/` → `repositories/` → `models/` with `schemas/` (Pydantic) as the request/response contract, plus `services/` for cross-cutting logic (notifications, allocation, metrics, storage, hanko). Routers take a `Session` via `Depends(get_db)` and instantiate a repository; business logic lives in repositories/services, not in route handlers.
- **Config**: `app/core/config.py` exposes a single `settings` instance (`pydantic-settings`, reads `.env`). Key env vars: `APP_DOMAIN`, `APP_DOMAIN_URL`, `APP_DATABASE_DSN`, `CORS_ALLOW_ORIGINS`, `REDIS_URL`, `HANKO_API_URL`, `HANKO_API_KEY`, `HANKO_AUDIENCE`, `SUPERADMIN_EMAIL`, `RESEND_API_KEY`, `NOTIFICATION_FROM_EMAIL`, `STORAGE_BACKEND` + `S3_*`. Superadmins are config-defined via `SUPERADMIN_EMAIL` (comma-separated, matched case-insensitively against the Hanko-verified email), never stored in the database — users have no global role column; per-org roles live on membership rows.
- **Database**: `app/core/database.py` builds a SQLAlchemy engine from `APP_DATABASE_DSN`. Defaults to SQLite for local dev but production uses PostgreSQL (`psycopg2`). Alembic migrations live in `app/alembic/` and are excluded from all linters. `app/core/audit.py` registers SQLAlchemy event listeners that write audit logs.
- **Auth**: `app/core/auth.py` validates Hanko-issued JWTs using a cached `PyJWKClient` against `{HANKO_API_URL}/.well-known/jwks.json`, algorithm `RS256`, audience = `HANKO_AUDIENCE`. `get_current_user` returns the decoded payload dict; there is no local user session.
- **Notifications**: routers/repositories never send email directly — they call a `notify_<event>()` helper in `app/services/notifications.py`, which writes in-app notification rows and fans out to channels in `app/services/channels/` (email channel delivers via Resend hosted templates, pushed from `apps/emails/`). Email delivery is off until `RESEND_API_KEY` is set; everything else still works.
- **Background jobs**: arq worker defined in `app/worker.py` (`WorkerSettings`); the queue name is `settings.APP_DOMAIN`. Enqueue helpers live in `app/tasks.py` — `enqueue_task` opens and closes a fresh pool per call. Register new task functions in `WorkerSettings.functions`; cron jobs (e.g. `cron_mark_overdue_capital_calls`) in `WorkerSettings.cron_jobs`.

#### Coding rules

- All models that have primary keys should have a `pydantic.UUID4` primary key field.
- Use repository pattern for business logic in routers and services.
- Worker and Task functions should use repository pattern.
- After creating new repository, schema, model, service, task - update app/*/__init__.py file with new imports.
- Define dependencies for router in router file itself, not in main.py file.

### Frontend SPAs (`apps/{manager,investor,superadmin}/src/`)

- **Stack**: React 18 + Vite 7 + TypeScript, Tailwind CSS v4 (via `@tailwindcss/vite`), React Router v6, TanStack Query, Radix UI primitives. Shared components come from `@edenscale/ui`; app-specific ones live in each app's `components/`.
- **Structure**: each app has `App.tsx` (routes), `layouts/`, `pages/`, `hooks/`, `lib/`. API access, auth, and org context come from the `@edenscale/*` packages — don't duplicate them per app.
- **Path alias**: `@/*` resolves to `src/*` (configured in both `vite.config.ts` and `tsconfig.json`).

### Contract between backend and frontend

The OpenAPI schema is the source of truth. When you change any route, request model, or response model in the backend, you must run `make openapi` before committing so `apps/backend/openapi.json` and `packages/api/src/schema.d.ts` stay aligned. Frontend type errors after backend changes almost always mean this step was skipped.
