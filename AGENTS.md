# AGENTS.md

This file gives coding agents the context needed to work safely in this repository.

## Project Shape

EdenScale is a two-service monorepo:

- `apps/backend/` - FastAPI, Python 3.12, SQLAlchemy, Alembic, arq.
- `apps/manager/` - React/Vite manager app mounted at `/manager`.
- `apps/investor/` - React/Vite investor app mounted at `/investor`.
- `packages/api/`, `packages/auth/`, `packages/shared/`, `packages/ui/` - shared frontend workspace packages.
- `apps/design/` - NewTaven brand assets, fonts, design tokens, and reference UI.
- `docs/architecture/` - architecture references for backend layering, RBAC, routing, and schema.

The backend and frontend are coupled through generated OpenAPI types. Do not hand-write frontend API types.

## Common Commands

Run these from the repository root unless noted otherwise:

- `make sync` - install backend dependencies and root pnpm workspace dependencies.
- `make start-backend` - start FastAPI on `localhost:8000`.
- `make start-manager` - start manager Vite on `0.0.0.0:3000`.
- `make start-investor` - start investor Vite on `0.0.0.0:3001`.
- `make start-frontend` - legacy alias for `make start-manager`.
- `make start-worker` - start the arq worker.
- `make test` - run backend pytest suite.
- `make lint` - run backend import smoke test, ruff, ty, black, and isort.
- `make openapi` - regenerate `apps/backend/openapi.json` and `packages/api/src/schema.d.ts`.
- `make migration` - create an Alembic migration.
- `make upgrade` / `make downgrade` - apply or revert migrations.

Useful targeted commands:

- `cd apps/backend && uv run pytest tests/test_api.py::test_name -v`
- `pnpm turbo run lint`
- `pnpm turbo run typecheck`
- `pnpm turbo run build --filter=manager`
- `pnpm turbo run build --filter=investor`

## Required Checks

Before considering a change complete:

1. Run the narrowest relevant test or type check while iterating.
2. Run `make test` for backend behavior changes.
3. Run `make lint` for backend code changes.
4. Run `make openapi` after any backend route, request schema, or response schema change.
5. Run `pnpm turbo run lint` and `pnpm turbo run typecheck` for frontend TypeScript changes.

If a command cannot be run in the current environment, report that clearly with the reason.

## Backend Conventions

Backend code lives in `apps/backend/app/`.

- `app/main.py` creates the FastAPI app and mounts routers.
- `routers/` owns HTTP shape, dependencies, RBAC gates, and error mapping.
- `repositories/` owns database queries, scoping, state transitions, and domain rules.
- `models/` owns SQLAlchemy table mappings and relationships.
- `schemas/` owns Pydantic request and response DTOs.
- `services/` owns cross-cutting domain services such as allocation, storage, Hanko, and notifications.

Keep route handlers thin. Do not put raw SQL, ORM construction, or business rules directly in routers when a repository or service is the right layer.

Authentication uses Hanko JWTs. Authorization is handled through `app/core/rbac.py` and repository-level scoping. Preserve role-aware filtering when adding list/read endpoints.

Alembic migrations live in `apps/backend/app/alembic/`. Use migrations for schema changes; `make db-init` is dev-only.

## Frontend Conventions

Frontend app code lives in `apps/manager/src/` and `apps/investor/src/`.

- `App.tsx` defines each application's routes.
- `layouts/` contains shell and page layout structure.
- `packages/ui/src/components/ui/` contains shadcn-style reusable primitives.
- `components/<domain>/` contains feature-specific UI.
- `hooks/` contains React hooks for auth, active organization, API queries, navigation, and UI state.
- `packages/api/src/client.ts` exposes the typed `openapi-fetch` client.
- `packages/api/src/schema.d.ts` is generated by `make openapi`; do not edit it manually.

Use the `@/*` alias for app-local imports from `src/*`. Use `@edenscale/*` package exports for shared API/auth/UI/utilities.

When changing backend API contracts, regenerate the OpenAPI client before fixing frontend call sites. Frontend type errors after a backend change usually mean the generated schema is stale.

## Design Guidance

For branded UI work, consult `apps/design/SKILL.md` and `apps/design/README.md`.

Default to the existing NewTaven design system:

- Cormorant Garamond for display type.
- Inter Tight for interface text.
- Conifer brand colors, parchment backgrounds, and ink text.
- Hairline rules over heavy shadows.
- Square or lightly rounded corners.
- Lucide icons where an icon is appropriate.

Match existing production UI patterns before introducing new ones.

## Agent Workflow

- Inspect existing code and docs before editing.
- Keep changes scoped to the requested behavior.
- Do not overwrite generated files by hand.
- Do not revert unrelated user changes.
- Prefer existing repository patterns over new abstractions.
- Add focused tests for behavior changes and bug fixes.
- For data model changes, include an Alembic migration and update affected tests.
- For API contract changes, regenerate OpenAPI and commit both generated artifacts.
