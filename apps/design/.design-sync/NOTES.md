# design-sync notes — apps/design

## What this repo actually is

`apps/design` has no `package.json`/build of its own. It's a **tokens-only** design
system: `colors_and_type.css` (tokens, semantic type classes, `@font-face`),
`fonts/`, and a few hand-authored reference files (`ui_kits/website/*.jsx`,
`preview/*.html`). The `ui_kits/website/*.jsx` components take **zero props** and
hardcode all their copy — they're illustrative demos, not real reusable
components. Do not try to sync them as real components without first adding
props, a real build, and a `.d.ts` contract — that would fabricate an API
contract that doesn't exist.

## Synthetic package for the converter

The converter needs a `package.json` + entry to run at all. Created:
- `package.json` at the `apps/design` root (`name: newtaven-design-system`)
- `.design-sync/synth-entry.mjs` — deliberately empty (`export {};`)

`cfg.cssEntry: "colors_and_type.css"` is what actually ships the tokens/type/fonts.
Build/validate deps (`esbuild`, `ts-morph`, `@types/react`, `react`, `react-dom`,
`playwright@1.61.1` — pinned to match the cached `chromium-1228` build) live in
`.ds-sync/node_modules`, passed as `--node-modules`. None of this is a real
dependency tree — it exists only to satisfy the converter.

## Rebranding context (important for future syncs)

The remote project is named **"EdenScale Design System"** — EdenScale is this
repo's real company/brand name. `apps/design` locally is a **rebrand**:
someone find-replaced "EdenScale" → "NewTaven" throughout (tokens header,
README, `Hero.jsx` copy, logo filenames `edenscale-*.svg` → `newtaven-*.svg`) to
use as a fictional private-equity brand for the `apps/web` marketing site demo.

The user explicitly confirmed (2026-07-01) that they want the cloud project's
**content** overwritten with this NewTaven-branded version — this was a
deliberate choice, not a default. The project's **name** on claude.ai/design is
still "EdenScale Design System" (no rename capability in the `DesignSync` tool);
only the content is NewTaven-branded now. Flag to the user if they want to
rename the project itself in the claude.ai/design UI.

## JetBrains Mono webfont

`--font-mono` referenced "JetBrains Mono" as a fallback with no shipped
webfont (`[FONT_MISSING]`). User explicitly chose to ship it for real rather
than accept the system-font substitute. Added `fonts/JetBrainsMono-400.woff2`
(Regular weight, Latin subset only — matches actual usage, small UI/eyebrow
labels never use other weights) sourced from Google Fonts' CDN
(`fonts.gstatic.com`, SIL Open Font License), plus the matching `@font-face` in
`colors_and_type.css`. If other weights are ever used for mono text, source
and add those too rather than assume they're covered.

## Loose extras synced outside the converter's own contract

`preview/**` (17 token/brand specimen cards), `ui_kits/website/**` (9 demo
files), `SKILL.md`, and root `colors_and_type.css` are **not** part of this
skill's component-bundle contract (no `_ds_sync.json` anchor covers them) —
they were pushed verbatim because the remote project already used this
convention before any design-sync run touched it, and local files
(`preview/_base.css`, `ui_kits/website/index.html`) depend on
`colors_and_type.css` existing at the project root.

**Re-sync risk**: because these aren't anchored, a future re-sync's diff
won't know if they've changed — if `apps/design/preview/**` or
`apps/design/ui_kits/**` or `SKILL.md` or root `colors_and_type.css` are
edited, manually re-push them (they're cheap, just re-`write_files` the same
paths). `_adherence.oxlintrc.json` and `_ds_manifest.json` are server-owned
(regenerated by the app's self-check) — never write or delete those.
